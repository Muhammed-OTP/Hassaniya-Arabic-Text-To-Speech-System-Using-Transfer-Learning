Introduction

Motivation & problématique

La synthèse vocale (TTS) permet de convertir du texte en parole naturelle — essentielle pour l'accessibilité, l'éducation et les assistants vocaux.

Fracture numérique linguistique

Les dialectes arabes sous-représentés manquent d'outils vocaux. L'arabe standard (MSA) domine les systèmes existants — les locuteurs hassaniya n'ont presque aucun TTS adapté.

Applications concrètes

Lecteurs vocaux pour malvoyants, applications éducatives en dialecte, préservation du patrimoine oral, interfaces vocales pour services publics en Mauritanie.

Contrainte du projet

Seulement 294 échantillons disponibles. Entraîner un TTS from scratch est impossible — nous adoptons une approche MVP / preuve de concept par apprentissage par transfert.

Contexte

Le dialecte hassaniya

Dialecte arabe parlé principalement en Mauritanie, Sahara occidental, Mali et Sénégal — langue à ressources très limitées (low-resource).

Aire géographique

~4 millions de locuteurs. Dialecte bedouin arabe, influencé par l'amazigh et des substrats africains. Très présent dans la vie quotidienne mauritanienne.

Pas d'orthographe standardisée

Le même mot peut s'écrire de plusieurs façons (Ø£/Ø¥/Ø¢ pour Alef). Notre pipeline normalise ces variantes avant la tokenisation.

Ressources numériques rares

Peu de corpus annotés, pas de modèles TTS/ASR publics. Proche phonologiquement de l'arabe standard — base pour le transfert de connaissances.

Annotation

Annotation & prétraitement du texte

Normalisation des variantes orthographiques et tokenisation au niveau des caractères pour alimenter le modèle TTS.

Normalisation Unicode

NFKC, suppression diacritiques, unification Alef (Ø£ Ø¥ Ø¢ → Ø§), nettoyage ponctuation, espaces normalisés.

Tokenisation caractère

Espaces → <space>. Vocabulaire : <pad>, <sos>, <eos> + ~39 caractères arabes.

Architecture

Architecture du modèle TTS

Encodeur de texte → décodeur Mel → vocodeur — implémentation PyTorch démontrée dans le notebook 04.

Encodeur (Tacotron2-style)

Embedding caractères → 3× Conv1D → BiLSTM. Extrait les représentations phonétiques du texte hassaniya.

Décodeur Mel

LSTM autorégressif → projection linéaire → spectrogramme Mel (80 bandes) + stop token.

Vocodeur HiFi-GAN

Pré-entraîné, convertit Mel en onde. Non affiné dans ce MVP — prévu pour phase 2.

class HassaniyaTTS(nn.Module):
  def __init__(self, vocab_size):
    self.encoder = HassaniyaTTSEncoder(vocab_size)
    self.decoder = HassaniyaTTSDecoder()
  def forward(self, text_ids):
    enc = self.encoder(text_ids)
    mel, stop = self.decoder(enc)
    return mel, stop

Discussion

Défis & limitations

Contraintes réelles d'un dialecte low-resource — documentées avec transparence.

Peu de données

294 échantillons vs 5 000–10 000 h pour production. Risque de surapprentissage si affinage agressif.

Orthographe non standard

Variantes d'écriture pour le même mot. La normalisation aide mais ne résout pas toute l'ambiguïté.

Pas de modèle hassaniya

Aucun TTS/ASR public. gTTS produit de l'arabe MSA — accent et prosodie incorrects.

Contraintes GPU

Affinage Tacotron2 nécessite GPU Colab (T4). Non réalisé dans le délai du projet.

Qualité audio variable

Enregistrements non studio — bruit de fond, niveaux inconsistants.

Délai d'une nuit

Priorité au pipeline documenté et à la compréhension — pas à l'entraînement long.
